Scenario Project Requirements

For this project, you need to develop a secure online voting system that can be used by small and medium organizations such as high schools, colleges, and corporations.

The basic model will require a login system where the administrators of the election can set up basic parameters such as the number of candidates, whether they want to show the election live, and when to announce the winner.

It should also include a security system for the voters. They must confirm their identity and there should be dual verification either from their phone numbers or registered email.

The more advanced version will integrate AI for any suspicious votes and notify the administrators if need be. It could also collect relevant data such as religion, gender, ethnicity, region, occupation etc and then share this data so that future candidates can learn who is supporting them.

Functional Requirements
Requirement Priority Justification
System should provide a login. High This is required for users to access the system to cast their votes.
System should allow users to cast their votes. High This is the base functionality of the project.
System must store the votes somewhere e.g: a server. High This is necessary to ensure that the votes can be properly counted and verified.
System should allow administers to set up basic parameters. Medium Some aspects are necessary to establish for the voting to run properly, others are not.

Non-Functional Requirements
Requirement Priority Justification
System should be secure. High The security being high will ensure the validity of the votes that have been entered.
System should detect suspicious votes. High This is helpful to ensure that fraudulent votes are not counted.
System should remove suspicious votes. Medium This is useful for keeping the ballots clean of fake votes, but they could be just identified and ignored instead.
System should collect user data. Medium The data could be used by candidates for more targeted marketing. The audience must be informed that their data is being taken and what exact data it is.
System should share user data with participants. Medium Allows candidates to make judgements using data but it could be given on request as opposed to having it all be sent.
